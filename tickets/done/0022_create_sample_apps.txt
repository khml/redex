チケット番号: 0022
タイトル: サンプルアプリケーション例の作成

概要:
Redex ライブラリを実際のアプリケーション内で活用するイメージを示すサンプルコード群を作成する。単なる式評価例ではなく、現実的なユースケース（CLI、Web 統合、Rake タスク、外部コンテキスト連携など）での利用例を用意し、利用者が導入しやすいドキュメントとテストを含める。

目的:
- ライブラリの利用シナリオを示し採用ハードルを下げる
- API の使い方（高レベル/低レベル）を具体例で示す
- ドキュメントとテストを整備してメンテしやすくする

サンプル候補（各サンプルは examples/ 配下にファイルを作成）:
- CLI 電卓アプリ: examples/cli_calculator.rb — 標準入力/引数で式を受け取り評価、履歴表示、簡易REPL
- Sinatra/小規模Web API: examples/sinatra_demo.rb — HTTP 経由で式を評価して JSON を返す（入力バリデーション含む）
- Rake タスク連携: examples/rake_task.rb — ビルド時に式を評価して設定値を生成する例
- context/ruby_resolver 利用例: examples/context_and_resolver.rb — 外部コンテキストや Ruby コールバックで未解決識別子を動的に解決する例
- Rails/コントローラ統合（ドキュメント例）: 利用上の注意やセキュリティ（入力サニタイズ）を示す短いガイド

受け入れ基準:
- 各サンプルは README に短い紹介と実行方法を記載していること
- bundle exec rake または bundle exec rspec で関連する spec が実行できる（最小限の動作確認用テスト）
- サンプル実行に必要な依存は極力既存 Gemfile で賄う（必要なら :development セクションに追加）
- セキュリティ面（外部入力の扱い）について短い注意書きがあること

作業タスク:
1. サンプル要件を各サンプルごとに詳細化してドキュメント化（担当: 調査）
2. examples/ ディレクトリ作成とサンプルスクリプト追加（担当: 実装）
3. README へサンプル一覧と実行手順を追記（担当: ドキュメント）
4. 最小の spec を追加して CI/ローカルで実行確認（担当: テスト）
5. コードレビューとマージ準備（担当: レビュー）

見積り: 小規模サンプル3点で 3-5 時間、包括的だと 1 日程度

備考:
- サンプルは教育目的のため、潜在的なセキュリティリスク（任意コード実行等）を README に明示すること
- まずは CLI と context/resolver の 2 つを優先実装する提案
