タイトル: README.md の充実化

概要:
プロジェクトのトップドキュメントである `README.md` を整備し、リポジトリを初めて見る開発者や利用者に必要な情報（目的、機能、セットアップ、使い方、参照ドキュメント、開発フロー）を分かりやすく提供する。

目的:
- 本ライブラリの目的と主要機能を明確にする。
- ローカルでのセットアップ手順とテスト実行手順を示す。
- 主要な設計ドキュメント（`docs/`）への導線を提供する。
- 貢献方法やライセンス、連絡先を明記する。

背景:
現在 `docs/` に詳細な設計や要求仕様が揃っているが、`README.md` にそれらの要約や参照がほとんどないため、新規参入者がプロジェクトを理解しづらい状態である。

受け入れ基準:
- `README.md` がリポジトリのルートに存在し、以下の項目を含むこと:
  - プロジェクト概要（1段落）
  - 主要機能の箇条書き
  - 必要環境（Rubyバージョン等）とセットアップ手順（依存インストール、テスト実行コマンド）
  - 簡単な使用例（最小限のサンプル）
  - 主要ドキュメントへのリンク（`docs/` 内の主要ファイル）
  - 貢献方法と連絡先（ISSUE/PRの作法）
  - ライセンス情報
- 既存の `spec/` を使ってテストが実行できるコマンド例を記載すること。
- リンケージ（`docs/` ファイルへの相対リンク）が動作すること。

作業項目:
1. `docs/` を読み、README に載せるべき要点を抽出する。
2. ローカルでセットアップ手順（Bundler / Ruby バージョン）とテスト手順を検証する。
3. `README.md` を作成／更新し、上記受け入れ基準を満たすコンテンツを追加する。
4. 簡単なコード例（Hello world もしくは簡単な評価例）を追加する。
5. 変更をコミットし、PR を作成する（PR テンプレートがあれば従う）。
