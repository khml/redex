チケット: 0020_add_float_literals

概要:
- `Tokenizer` と `Parser/Evaluator` に浮動小数点リテラル（例: `3.14`, `0.5`）のサポートを追加する。
- 現状: `lib/redex/tokenizer.rb` は整数のみをパースしているため、仕様（整数/浮動小数点両対応）と齟齬がある。

影響度: 中

関連ファイル:
- lib/redex/tokenizer.rb
- lib/redex/parser.rb
- lib/redex/evaluator.rb
- spec/tokenizer_spec.rb
- spec/parser_spec.rb

要件/受け入れ基準:
- トークナイザが浮動小数点リテラルを `:number` トークンとしてパースし、値は `Float` あるいは `Integer`（小数部がない場合は整数のままでも可）で返す。
- `3.14 + 1` → `4.14` が評価できる。
- `0.5 * 2` → `1.0` が評価できる。
- 既存の整数パースは壊さない。
- 既存のテストが存在する場合はテストを追加/更新してカバレッジを確保する。

実装案:
1. `Tokenizer.tokenize` の数値判定を拡張して浮動小数点（小数点 `.` を含む）を認識する。正規表現で `\A[0-9]+(\.[0-9]+)?` のようにマッチさせる。
2. 数値トークンの `value` を、整数のみの場合は `Integer`、小数部がある場合は `Float` を格納する（または一律 Float にする方針でも可）。
3. 必要に応じて `evaluator` の演算（特に除算）における数値型の扱いを見直す（整数除算が問題となる場合は `Float` 化を行う）。
4. `spec/tokenizer_spec.rb` と `spec/parser_spec.rb` にそれぞれ浮動小数点に関するテストケースを追加する。

テスト例:
- トークナイズ: `Tokenizer.tokenize("3.14 + x")` が `:number(3.14)`, `:op(+)`, `:ident(x)` を返す
- 評価: `Redex::Interpreter.evaluate("3.14 + 1")` が `4.14` を返す（または `result: 4.14` などの形式）
